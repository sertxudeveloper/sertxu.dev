Project Helios

Cloud infrastructure monitoring platform with real-time observability and alerting.

Live Demo Source Code
H

Project Helios is a real-time infrastructure monitoring platform built for teams that need instant visibility into their cloud environments. It aggregates metrics, logs, and events from multiple providers into a single dashboard with customizable alerting rules.

Architecture Overview

The platform uses a modular agent-based architecture. Lightweight agents are deployed to each target environment and communicate back to a central aggregator via secure WebSocket connections. This design minimizes latency and keeps data collection reliable even under network interruptions.

Here's a simplified example of the agent's health check reporter: 

func (a *Agent) ReportHealth(ctx context.Context) (*Health, error) {
    checks := []Check{
        {Name: "cpu", Fn: a.checkCPU},
        {Name: "memory", Fn: a.checkMemory},
        {Name: "disk", Fn: a.checkDisk},
        {Name: "uptime", Fn: a.checkUptime},
    }

    results := make([]Result, 0, len(checks))
    for _, c := range checks {
        result, err := c.Fn(ctx)
        if err != nil {
            log.Printf("check %s failed: %v", c.Name, err)
        }
        results = append(results, result)
    }

    return &Health{
        AgentID:   a.id,
        Timestamp: time.Now().UTC(),
        Checks:    results,
    }, nil
}

Key Features

  • Multi-cloud support — AWS, GCP, and Azure metrics aggregated under a single pane of glass
  • Custom alert rules — Define thresholds, notification channels, and escalation policies through a drag-and-drop editor
  • Real-time dashboards — WebSocket-backed live updates with sub-second latency for critical metrics
  • Team collaboration — Share dashboards, annotate incidents, and integrate with Slack, PagerDuty, and email
  • Historical analysis — Query and visualize up to 12 months of retained metrics with automatic downsampling
"We reduced our mean time to detection from 12 minutes to under 30 seconds after deploying Helios. The custom alert rules alone saved us from three major outages in the first month."

Tech Stack Decisions

The backend is written in Go for its performance characteristics and excellent concurrency model. Each agent runs as a single static binary with no external dependencies, making deployment as simple as copying a file.

Why Go for the Agent

Go's goroutines map naturally to the concurrent check execution pattern. Each health check runs in its own goroutine with a configurable timeout, ensuring a single hung check never blocks the entire reporting cycle.

Frontend Architecture

The dashboard is built with React and uses D3 for custom visualizations. Real-time data flows through Server-Sent Events with automatic reconnection and backpressure handling. The component tree is structured around a central metrics store that deduplicates and normalizes incoming data before distribution to subscribers.

Project Helios is currently running in production across five environments and processes over 2 million health checks per day. The entire infrastructure is defined as Terraform modules and deployed through a CI/CD pipeline with automated canary analysis.
